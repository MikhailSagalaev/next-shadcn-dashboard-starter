# InSales Integration - Quick Start Guide

**Дата:** 2026-03-04  
**Статус:** ✅ Готово к тестированию

---

## 🚀 Быстрый старт

### 1. Открыть страницу интеграции

**Правильный URL:**
```
https://gupil.ru/dashboard/projects/[PROJECT_ID]/integrations/insales
```

**Пример:**
```
https://gupil.ru/dashboard/projects/cmilhq0y600099e7uraiowrmt/integrations/insales
```

⚠️ **Важно:** `integrations` (множественное число), не `integration`

---

### 2. Создать интеграцию

**Шаги:**
1. Открыть страницу интеграции
2. Заполнить форму:
   - **API Key:** Получить в InSales → Настройки → API
   - **API Password:** Получить там же
   - **Shop Domain:** `your-shop.myinsales.ru`
   - **Bonus Percent:** 5 (5% от суммы заказа)
   - **Max Bonus Spend:** 30 (максимум 30% от заказа)
3. Нажать "Сохранить настройки"

**Результат:**
- ✅ Интеграция создана
- ✅ Webhook URL сгенерирован
- ✅ Webhook Secret создан
- ✅ Код виджета готов

---

### 3. Настроить Webhook в InSales

**Шаги:**
1. Скопировать Webhook URL из админки Gupil
2. Открыть InSales → Настройки → Webhooks
3. Добавить новый webhook:
   - **URL:** Вставить скопированный URL
   - **События:** Выбрать:
     - `clients/create` - создание клиента
     - `orders/create` - создание заказа
     - `orders/update` - обновление заказа
4. Сохранить

**Формат Webhook URL:**
```
https://gupil.ru/api/insales/webhook/[PROJECT_ID]
```

---

### 4. Установить виджет на сайт

**Шаги:**
1. Скопировать код виджета из админки Gupil
2. Открыть InSales → Дизайн → Редактор кода
3. Найти файл `layout.liquid` или `theme.liquid`
4. Вставить код перед закрывающим тегом `</body>`
5. Сохранить

**Код виджета:**
```html
<script 
  src="https://gupil.ru/insales-widget-loader.js" 
  data-project-id="[PROJECT_ID]"
></script>
```

---

### 5. Протестировать

**Тест 1: Создание клиента**
1. Зарегистрировать нового клиента в InSales
2. Проверить в Gupil → Управление бонусами → Пользователи
3. Должен появиться новый пользователь
4. Если настроены приветственные бонусы → должны быть начислены

**Тест 2: Создание заказа**
1. Создать заказ в InSales (статус "Оплачен")
2. Проверить в Gupil → История транзакций
3. Должна появиться транзакция "Заказ #XXX"
4. Бонусы должны быть начислены (5% от суммы)

**Тест 3: Виджет**
1. Открыть сайт магазина
2. Виджет должен появиться в правом нижнем углу
3. Авторизоваться в магазине
4. Виджет должен показать баланс бонусов

**Тест 4: Применение бонусов**
1. Добавить товары в корзину
2. Перейти к оформлению заказа
3. В виджете ввести количество бонусов для списания
4. Нажать "Применить"
5. Промокод должен автоматически примениться
6. Итоговая сумма должна уменьшиться

## 🐛 Troubleshooting

### Проблема: 404 на странице интеграции

**Причина:** Неправильный URL

**Решение:** Использовать `integrations` (множественное число)
```
❌ /dashboard/projects/[id]/integration/insales
✅ /dashboard/projects/[id]/integrations/insales
```

### Проблема: Webhook не работает

**Проверить:**
1. Webhook URL правильный?
2. События выбраны (clients/create, orders/create)?
3. Интеграция активна в Gupil?
4. Проверить логи webhooks в админке

### Проблема: Бонусы не начисляются

**Проверить:**
1. Заказ оплачен (payment_status = 'paid')?
2. Процент начисления > 0?
3. Пользователь существует в системе?
4. Проверить логи webhooks

### Проблема: Виджет не загружается

**Проверить:**
1. Код виджета вставлен правильно?
2. data-project-id корректный?
3. Открыть консоль браузера (F12)
4. Проверить ошибки JavaScript

### Проблема: Промокод не применяется

**Проверить:**
1. Достаточно бонусов на балансе?
2. Не превышен maxBonusSpend (30%)?
3. API credentials InSales правильные?
4. Проверить логи в админке

```bash

  -H "Content-Type: application/json" \
  -d '{"id":"clients/create","client":{"id":12345,"email":"test@example.com"}}'
```
