# Super-Admin: Управление версиями виджета

**Путь:** `/super-admin/widget-versions`  
**Доступ:** Только для супер-администраторов  
**Статус:** ✅ Реализовано (2026-02-01)

---

## 🎯 Назначение

Страница для управления миграцией проектов с legacy виджета (`tilda-bonus-widget.js`) на universal архитектуру (`widget-loader.js` + `universal-widget.js`).

### Ключевые возможности
- Просмотр всех проектов с текущей версией виджета
- Статистика распределения версий
- Переключение версии для отдельных проектов
- Поиск и фильтрация проектов
- Мониторинг активности проектов

---

## 📊 Интерфейс

### Статистика (верхняя панель)

**4 карточки с метриками:**

1. **Всего проектов**
   - Общее количество проектов в системе
   - Иконка: Activity (синяя)

2. **Legacy виджет**
   - Количество проектов на legacy версии
   - Процент от общего числа
   - Иконка: Package (янтарная)

3. **Universal виджет**
   - Количество проектов на universal версии
   - Процент от общего числа
   - Иконка: Rocket (зеленая)

4. **Прогресс миграции**
   - Процент мигрированных проектов
   - Иконка: TrendingUp (фиолетовая)

### Таблица проектов

**Колонки:**
- **Проект** - название и домен
- **Владелец** - email владельца проекта
- **Пользователи** - количество зарегистрированных пользователей
- **Активность (7д)** - количество webhook событий за последние 7 дней
- **Создан** - дата создания проекта (относительная)
- **Версия виджета** - текущая версия (Legacy/Universal)
- **Действия** - кнопка переключения версии

**Функции:**
- 🔍 **Поиск** - по названию проекта, домену или email владельца
- 🎯 **Фильтр** - все версии / только legacy / только universal
- ⬆️⬇️ **Сортировка** - по любой колонке (клик на заголовок)

## 🔄 Переключение версии

### Процесс

1. **Нажать кнопку "Переключить"** в строке проекта
2. **Подтверждение** - появится диалог с информацией:
   - Название проекта
   - Целевая версия (Legacy → Universal или Universal → Legacy)
   - Предупреждение о последствиях
   - Информация о немедленном вступлении в силу
3. **Подтвердить** - нажать "Переключить" в диалоге
4. **Результат** - toast уведомление и автообновление страницы

### Информация в диалоге

**При переключении на Universal:**
```
Universal версия: Новая архитектура с поддержкой множества платформ.
Убедитесь, что проект протестирован перед переключением.
```

**При переключении на Legacy:**
```
Legacy версия: Откат на старую версию виджета.
Используйте только при проблемах с Universal версией.
```

### Что происходит при переключении

1. **База данных** - обновляется поле `widgetVersion` в таблице `projects`
2. **Логирование** - создается запись в `system_logs` с деталями изменения
3. **Немедленное действие** - виджет на сайте клиента использует новую версию при следующей загрузке страницы
4. **Прозрачность** - клиент не видит изменений, переключение происходит на сервере

## 📋 Примеры использования

### Сценарий 1: Внутреннее тестирование

**Цель:** Протестировать universal виджет на 1-2 проектах

**Шаги:**
1. Открыть `/super-admin/widget-versions`
2. Найти тестовый проект через поиск
3. Убедиться, что проект неактивен или имеет минимальную активность
4. Нажать "Переключить" → подтвердить
5. Проверить работу виджета на сайте проекта
6. Мониторить логи и метрики в течение 1-2 дней
7. При проблемах - переключить обратно на Legacy

### Сценарий 2: Бета-тестирование

**Цель:** Расширить тестирование на 5-10 активных проектов

**Шаги:**
1. Отфильтровать проекты по версии "Legacy"
2. Отсортировать по активности (7д)
3. Выбрать проекты со средней активностью
4. Связаться с владельцами (опционально)
5. Переключить проекты на Universal
6. Мониторить метрики ежедневно
7. Собирать feedback от клиентов

### Сценарий 3: Откат при проблемах

**Цель:** Быстро вернуть проект на Legacy при критических ошибках

**Шаги:**
1. Открыть `/super-admin/widget-versions`
2. Найти проблемный проект
3. Нажать "Переключить" (Universal → Legacy)
4. Подтвердить откат
5. Проверить восстановление работы
6. Зафиксировать проблему для исправления

### Сценарий 4: Массовая миграция

**Цель:** Переключить все стабильные проекты на Universal

**Шаги:**
1. Отфильтровать Legacy проекты
2. Отсортировать по дате создания (старые первыми)
3. Переключать по 10-20 проектов в день
4. Мониторить метрики после каждой волны
5. При стабильной работе - продолжать миграцию
6. Оставить проблемные проекты на Legacy

## 🔍 Мониторинг

### Метрики для отслеживания

**На странице:**
- Общее количество проектов
- Распределение по версиям (Legacy vs Universal)
- Прогресс миграции (%)

**В логах (SystemLog):**
```sql
SELECT * FROM "system_logs"
WHERE message LIKE '%Widget version changed%'
ORDER BY "createdAt" DESC;
```

**Активность по версиям:**
```sql
SELECT 
  p."widget_version",
  COUNT(DISTINCT w."projectId") as active_projects,
  COUNT(*) as webhook_events
FROM "webhook_logs" w
JOIN "projects" p ON p.id = w."projectId"
WHERE w."createdAt" > NOW() - INTERVAL '7 days'
GROUP BY p."widget_version";
```

**Ошибки по версиям:**
```sql
SELECT 
  p."widget_version",
  COUNT(*) as error_count
FROM "webhook_logs" w
JOIN "projects" p ON p.id = w."projectId"
WHERE w.success = false
  AND w."createdAt" > NOW() - INTERVAL '7 days'
GROUP BY p."widget_version";
```

## ⚠️ Важные замечания

### Безопасность
- ✅ Только супер-админы имеют доступ к странице
- ✅ Все изменения логируются с указанием администратора
- ✅ Клиенты не видят настройку версии
- ✅ Можно откатиться в любой момент

### Производительность
- ✅ Изменение вступает в силу немедленно
- ✅ Не требует перезапуска серверов
- ✅ Не влияет на работу других проектов
- ✅ Виджет обновляется при следующей загрузке страницы клиента

### Тестирование
- ⚠️ Всегда тестируйте на неактивных проектах сначала
- ⚠️ Мониторьте логи после переключения
- ⚠️ Будьте готовы к быстрому откату
- ⚠️ Информируйте владельцев активных проектов (опционально)

## 📞 Поддержка

**При проблемах:**
1. Проверить логи в `/super-admin/errors`
2. Проверить SystemLog для конкретного проекта
3. Откатить проект на Legacy версию
4. Зафиксировать проблему для команды разработки

**Emergency Rollback (все проекты):**
```sql
UPDATE "projects" SET "widget_version" = 'legacy';
```

**Kill Switch (.env):**
```env
FORCE_LEGACY_WIDGET=true
```
