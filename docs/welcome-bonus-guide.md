# 🎁 Руководство по приветственным бонусам

## Что это такое?

Приветственные бонусы - это автоматическое вознаграждение, которое получает каждый новый пользователь при регистрации в вашей бонусной программе.

## 🚀 Быстрая настройка

### 1. Откройте настройки проекта
В админ панели перейдите в раздел "Настройки проекта"

### 2. Найдите секцию "Приветственное вознаграждение"
Здесь вы можете настроить два типа вознаграждения:

#### Вариант A: Бонусы (рекомендуется)
- **Тип**: Бонусы
- **Сумма**: Например, 500 рублей
- **Как работает**: При регистрации пользователь получает 500 бонусов на счет

#### Вариант B: Скидка на первую покупку
- **Тип**: Скидка
- **Процент**: Например, 10%
- **Как работает**: При первой покупке пользователь получает скидку 10%

### 3. Сохраните настройки
Нажмите "Сохранить" - изменения вступят в силу немедленно

## 📊 Как это работает

### Автоматическое начисление

```
Пользователь регистрируется → Webhook → Создание пользователя → Автоматическое начисление бонусов
```

**Важно:** Workflow НЕ требуется! Бонусы начисляются автоматически при создании пользователя.

### Режимы работы

#### Режим WITH_BOT (с Telegram ботом)
1. Пользователь регистрируется через webhook
2. Создается аккаунт со статусом "Неактивен"
3. Приветственные бонусы начисляются, но **недоступны**
4. Пользователь привязывает Telegram → аккаунт активируется
5. Бонусы становятся **доступны для использования**

#### Режим WITHOUT_BOT (без Telegram бота)
1. Пользователь регистрируется через webhook
2. Создается аккаунт со статусом "Активен"
3. Приветственные бонусы начисляются и **сразу доступны**

## 🔧 Настройки

### Приоритет настроек

Система проверяет настройки в следующем порядке:

1. **ReferralProgram.welcomeBonus** (если реферальная программа активна)
2. **Project.welcomeBonus** (базовые настройки проекта)
3. Если оба = 0, приветственные бонусы не начисляются

### Условия начисления

Приветственные бонусы начисляются только если:
- ✅ `welcomeRewardType === 'BONUS'`
- ✅ `welcomeBonus > 0`
- ✅ Пользователь создается впервые (не существует в системе)

## 📝 Примеры

### Пример 1: Базовая настройка

**Настройки проекта:**
- Тип вознаграждения: Бонусы
- Сумма приветственных бонусов: 500

**Результат:**
- Каждый новый пользователь получает 500 бонусов при регистрации

### Пример 2: Скидка на первую покупку

**Настройки проекта:**
- Тип вознаграждения: Скидка
- Процент скидки на первую покупку: 15%

**Результат:**
- При первой покупке пользователь получает скидку 15%
- Бонусы на счет не начисляются

### Пример 3: Реферальная программа

**Настройки реферальной программы:**
- Приветственный бонус: 1000
- Тип: Бонусы

**Настройки проекта:**
- Приветственный бонус: 500

**Результат:**
- Используется значение из реферальной программы: 1000 бонусов

## 🐛 Решение проблем

### Бонусы не начисляются

**Проверьте:**
1. ✅ Тип вознаграждения = "Бонусы" (не "Скидка")
2. ✅ Сумма приветственных бонусов > 0
3. ✅ Пользователь создается впервые (не существует в базе)
4. ✅ Webhook отправляет корректные данные (email или phone)

**Проверка в логах:**
```
Создан новый пользователь → userId: xxx
Начислены приветственные бонусы → amount: 500
```

### Бонусы начислены, но недоступны

**Причина:** Режим работы проекта = `WITH_BOT`

**Решение:**
1. Пользователь должен привязать Telegram аккаунт через бота
2. После привязки аккаунт активируется
3. Бонусы становятся доступны

**Альтернатива:** Переключите проект в режим `WITHOUT_BOT` в настройках

### Бонусы начислились дважды

**Причина:** Пользователь был создан дважды (разные email/phone)

**Решение:**
- Система автоматически ищет пользователя по email и phone
- Убедитесь, что webhook отправляет одинаковые данные при каждой регистрации
- Проверьте нормализацию телефона (система автоматически приводит к формату +7XXXXXXXXXX)

## 📊 Мониторинг

### Просмотр начисленных бонусов

1. Перейдите в раздел "Пользователи"
2. Найдите нужного пользователя
3. Откройте детали → вкладка "Транзакции"
4. Найдите транзакцию типа "WELCOME"

### Статистика

В разделе "Статистика" вы можете увидеть:
- Общее количество зарегистрированных пользователей
- Сумму начисленных приветственных бонусов
- Процент активированных пользователей (для режима WITH_BOT)
