## ⚠️ Критические находки

### 1. TODO: Определение списанных бонусов
**Файл:** `src/lib/insales/insales-service.ts:116`  
**Проблема:** `const bonusSpent = 0; // TODO: получить из custom_fields заказа`

**Описание:**
- Логика BonusBehavior не работает корректно
- Всегда считается что бонусы НЕ использованы (bonusSpent = 0)
- Это означает что всегда начисляем на полную сумму

**Решение:**
1. InSales должен передавать информацию о списанных бонусах в webhook
2. Варианты:
   - Custom поле заказа `bonus_spent`
   - Анализ скидок (discount_amount)
   - Отдельный webhook event при применении промокода

**Приоритет:** 🔴 Высокий

**Временное решение:**
- Для тестирования можно использовать режим `SPEND_AND_EARN` (работает корректно)
- Или режим `EARN_ONLY` (списание бонусов отключено)

## 🧪 Команды для тестирования

### 1. Запуск автоматических тестов
```bash
bash test-insales.sh
```

### 2. Ручное тестирование webhook
```bash
# Отправить тестовый webhook
curl -X POST http://localhost:3000/api/insales/webhook/PROJECT_ID \
  -H "Content-Type: application/json" \
  -H "X-InSales-Signature: SECRET" \
  -d '{
    "id": "clients/create",
    "client": {
      "id": 12345,
      "name": "Test User",
      "email": "test@example.com",
      "phone": "+79991234567"
    }
  }'
```

### 3. Проверка баланса
```bash
curl http://localhost:3000/api/insales/balance/PROJECT_ID?email=test@example.com
```

### 4. Применение бонусов
```bash
curl -X POST http://localhost:3000/api/insales/apply-bonuses/PROJECT_ID \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "bonusAmount": 100,
    "orderTotal": 2000
  }'
```
