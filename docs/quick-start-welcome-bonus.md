# 🚀 Быстрый старт: Приветственные бонусы

## За 5 минут настройте автоматическое начисление бонусов новым пользователям

### Шаг 1: Откройте настройки проекта (30 сек)

1. Войдите в админ панель
2. Выберите ваш проект
3. Перейдите в "Настройки"

### Шаг 2: Настройте приветственное вознаграждение (1 мин)

Найдите секцию **"Приветственное вознаграждение при регистрации"**

#### Вариант A: Бонусы (рекомендуется)
```
Тип вознаграждения: [Бонусы ▼]
Сумма приветственных бонусов: [500] рублей
```

#### Вариант B: Скидка на первую покупку
```
Тип вознаграждения: [Скидка ▼]
Процент скидки на первую покупку: [10] %
```

### Шаг 3: Сохраните настройки (10 сек)

Нажмите кнопку **"Сохранить"** - изменения вступят в силу немедленно!

### Шаг 4: Проверьте работу (3 мин)

#### Способ 1: Тестовый webhook

```bash
# Отправьте тестовый запрос
curl -X POST https://your-domain.com/api/webhook/YOUR_SECRET \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Тест Пользователь",
    "email": "test@example.com",
    "phone": "+79001234567",
    "formid": "test_form",
    "tranid": "test_123"
  }'
```

#### Способ 2: Через админ панель

1. Перейдите в "Пользователи"
2. Найдите нового пользователя
3. Откройте детали → вкладка "Бонусы"
4. Проверьте наличие бонуса типа "WELCOME"

#### Способ 3: Автоматический тест

```bash
# Запустите тестовый скрипт
npx tsx scripts/test-welcome-bonus.ts
```

## ✅ Готово!

Теперь каждый новый пользователь автоматически получает приветственные бонусы при регистрации.

## 🔧 Дополнительные настройки

### Режим работы проекта

#### WITH_BOT (по умолчанию)
- Бонусы начисляются, но недоступны
- Требуется активация через Telegram
- Подходит для контроля регистраций

#### WITHOUT_BOT
- Бонусы начисляются и сразу доступны
- Telegram не требуется
- Подходит для быстрого старта

**Переключение:** Настройки проекта → Режим работы → Выберите нужный

### Реферальная программа

Если у вас активна реферальная программа, настройте приветственные бонусы там:

1. Перейдите в "Реферальная программа"
2. Укажите сумму приветственных бонусов
3. Эти настройки имеют приоритет над базовыми

## 🐛 Решение проблем

### Бонусы не начисляются?

**Проверьте:**
- ✅ Тип вознаграждения = "Бонусы"
- ✅ Сумма > 0
- ✅ Webhook отправляет email или phone

### Бонусы начислены, но недоступны?

**Причина:** Режим WITH_BOT

**Решение:**
1. Пользователь привязывает Telegram
2. Или переключите проект в режим WITHOUT_BOT

## 💡 Советы

### Оптимальная сумма бонусов

- **Малый бизнес:** 100-300 рублей
- **Средний бизнес:** 300-500 рублей
- **Крупный бизнес:** 500-1000 рублей

### Когда использовать скидку вместо бонусов

- Высокий средний чек (>5000₽)
- Разовые покупки (не требуется повторное возвращение)
- Премиум сегмент

### Когда использовать бонусы

- Низкий/средний чек (<5000₽)
- Повторные покупки (стимулирование лояльности)
- Массовый сегмент
