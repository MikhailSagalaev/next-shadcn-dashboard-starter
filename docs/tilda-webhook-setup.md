# 🔗 Настройка Webhook в Tilda

## Проблема: [400] Bad request: Invalid JSON

Если вы получаете ошибку `{"error":"Invalid JSON"}` при настройке webhook в Tilda, это означает, что система не может распарсить данные от Tilda.

### ✅ Решение

Проблема была в том, что **Tilda отправляет данные в формате `application/x-www-form-urlencoded`**, а не JSON. Система теперь поддерживает оба формата.

## 📋 Пошаговая настройка

### 1. Получите Webhook URL

В админ панели SaaS Bonus System:
1. Откройте настройки вашего проекта
2. Скопируйте Webhook Secret
3. Ваш webhook URL будет:
   ```
   https://your-domain.com/api/webhook/YOUR_WEBHOOK_SECRET
   ```

### 2. Настройте форму оплаты в Tilda

1. Откройте редактор страницы Tilda
2. Выберите блок с формой оплаты (обычно T706 - Корзина)
3. Перейдите в настройки блока → "Оплата и доставка"
4. Найдите раздел "Webhook для уведомлений"

### 3. Добавьте Webhook URL

В поле "URL для уведомлений" вставьте ваш webhook URL:
```
https://your-domain.com/api/webhook/YOUR_WEBHOOK_SECRET
```

### 4. Проверьте настройки

Tilda автоматически отправит тестовый запрос. Вы должны увидеть:
- ✅ Зеленую галочку, если webhook работает
- ❌ Ошибку, если что-то не так

## 🧪 Тестирование

### Вариант 1: Через Tilda
1. Сделайте тестовый заказ на вашем сайте
2. Проверьте логи webhook в админ панели

### Вариант 2: Через тестовую страницу
1. Откройте `https://your-domain.com/test-tilda-webhook.html`
2. Вставьте ваш webhook URL
3. Нажмите "Отправить Webhook"
4. Проверьте логи

## 📦 Формат данных от Tilda

Tilda отправляет данные в формате `application/x-www-form-urlencoded`:

```
name=Иван Иванов
email=test@example.com
phone=+79991234567
payment={"amount":"5000","orderid":"ORDER-123","products":[...]}
appliedBonuses=0
```

Обратите внимание: поле `payment` передается как **JSON-строка**.

## 🔍 Отладка

### Проверка логов webhook

В админ панели:
1. Перейдите в раздел "Webhook Logs"
2. Найдите последние запросы от Tilda
3. Проверьте:
   - `status`: должен быть 200
   - `success`: должен быть true
   - `body`: должен содержать данные заказа

### Типичные ошибки

#### ❌ Invalid JSON
**Причина:** Старая версия системы не поддерживала form data  
**Решение:** Обновите систему до последней версии

#### ❌ Invalid webhook secret
**Причина:** Неправильный webhook secret в URL  
**Решение:** Проверьте webhook secret в настройках проекта

#### ❌ User not found
**Причина:** Пользователь с таким email/phone не зарегистрирован  
**Решение:** Убедитесь, что пользователь сначала регистрируется через виджет
