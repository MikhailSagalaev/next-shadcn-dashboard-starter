## 🎯 Зачем мы это делаем?

### Главная цель
**Превратить Gupil из "бонусной системы для Tilda" в универсальную платформу лояльности для ЛЮБЫХ сайтов.**

### Ключевые проблемы, которые решаем:

1. **🔒 Привязка к одной платформе**
   - Сейчас работаем только на Tilda (~100 тыс. сайтов)
   - Теряем 80% потенциального рынка
   - Конкуренты с мультиплатформенностью выигрывают

2. **🧪 Невозможность тестировать**
   - Нет unit тестов (покрытие 0%)
   - Тестирование только вручную
   - Страх вносить изменения
   - Баги в production

3. **🔧 Сложность поддержки**
   - Монолит 2000+ строк
   - Все смешано в одном файле
   - Новый разработчик разбирается неделю
   - Добавление платформы = переписывание с нуля

4. **⏱️ Медленная разработка**
   - Новая фича = 6 часов работы
   - Добавление платформы = 2 недели
   - Много времени на баги

## 💡 Что дает новая архитектура?

### 1. Мультиплатформенность 🌐
```
Было: 1 платформа (Tilda)
Стало: Любые платформы (Tilda, Shopify, WooCommerce, Custom)

Рынок: 100 тыс. → 10+ млн сайтов (x100)
```

### 2. Тестируемость ✅
```
Было: 0% покрытие тестами
Стало: 80%+ покрытие

Баги: -80%
Уверенность в изменениях: +100%
```

### 3. Простота поддержки 🔧
```
Было: 2000 строк монолита
Стало: 600 строк Core + 300 строк адаптер

Время разбора кода: 1 неделя → 1 день
```

### 4. Быстрая разработка 🚀
```
Новая фича: 6 часов → 1 час (x6 быстрее)
Новая платформа: 2 недели → 1 день (x10 быстрее)
```

   - Автоопределение платформы
