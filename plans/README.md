1. **001** first — highest single-fix value, independent, stops double-payments now.
2. **002** — foundation for reliable payouts; touches the same `processReferralBonus` method as 001, so land 001 first to avoid merge churn.
3. **003** — refund reversal; reuses the deterministic `externalId` from 001 and the observable-failure pattern from 002.
4. **004** — cheap consistency fix; do before 005 so depth semantics are settled before chain resolution is rewritten.
5. **005** — removes the wrong-manager payout risk; larger, may need a migration.
6. **006** — design spike for the biggest product gap (money-out for partners); output is a design doc that becomes a future build plan (007).
