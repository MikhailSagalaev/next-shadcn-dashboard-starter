# Анализ UX: Настройка процента начисления бонусов

## 📊 Текущая ситуация

### Как работает сейчас

Система поддерживает **два режима** начисления бонусов:

#### 1. Простой режим (без уровней)
```
Project.bonusPercentage = 5%
```
- Все пользователи получают фиксированный процент
- Настройка в одном месте: "Настройки проекта" → "Процент бонусов"
- Просто и понятно

#### 2. Продвинутый режим (с уровнями)
```
BonusLevel[]:
  - Базовый: 0-10,000₽ → 3% начисление, 50% оплата
  - Серебряный: 10,000-50,000₽ → 5% начисление, 70% оплата
  - Золотой: 50,000+₽ → 10% начисление, 100% оплата
```
- Процент зависит от суммы покупок пользователя
- Настройка в отдельном разделе: "Уровни бонусов"
- Гибко, но сложнее

### Проблемы текущего UX

#### ❌ Проблема 1: Неочевидное переключение режимов
```tsx
{!hasLevels ? (
  <Input id='bonusPercentage' ... />
) : (
  <div>Процент определяется уровнями бонусов</div>
)}
```

**Что не так:**
- Пользователь не понимает, что есть два режима
- Непонятно, как переключиться между режимами
- Поле процента просто исчезает при создании уровней

#### ❌ Проблема 2: Разрозненные настройки
```
Настройки проекта (project-settings-view.tsx)
  └── Процент бонусов (только если нет уровней)

Уровни бонусов (bonus-levels-view.tsx)
  └── Список уровней с процентами
```

**Что не так:**
- Настройки в разных местах
- Нет единой точки входа
- Сложно понять связь между настройками

#### ❌ Проблема 3: Нет визуального переключателя
```
[ ] Простой режим (фиксированный процент)
[ ] Продвинутый режим (уровни бонусов)
```

**Что не так:**
- Нет явного выбора режима
- Режим определяется наличием уровней (неявно)
- Нельзя быстро переключиться

#### ❌ Проблема 4: Потеря данных при переключении
```
Сценарий:
1. Пользователь настроил 5 уровней
2. Удалил все уровни
3. Вернулся к простому режиму
4. Старые настройки уровней потеряны
```

**Что не так:**
- Нет предупреждения о потере данных
- Нельзя временно отключить уровни
- Нет возможности вернуться к предыдущим настройкам

## 💡 Предложения по улучшению

### Вариант 1: Явный переключатель режимов (Рекомендуется)

#### Визуальный дизайн
```tsx
┌─────────────────────────────────────────────────────────┐
│ Настройка начисления бонусов                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│ Выберите режим начисления:                              │
│                                                          │
│ ┌──────────────────┐  ┌──────────────────┐             │
│ │ ○ Простой режим  │  │ ● Уровни бонусов │             │
│ │                  │  │                  │             │
│ │ Фиксированный    │  │ Процент зависит  │             │
│ │ процент для всех │  │ от суммы покупок │             │
│ └──────────────────┘  └──────────────────┘             │
│                                                          │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ Настройки выбранного режима:                        │ │
│ │                                                      │ │
│ │ [Динамический контент в зависимости от режима]      │ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

#### Преимущества
- ✅ Явный выбор режима
- ✅ Понятно, что есть два варианта
- ✅ Все настройки в одном месте
- ✅ Легко переключаться

#### Реализация
```tsx
// Новое поле в Project
bonusMode: 'SIMPLE' | 'LEVELS'

// UI компонент
<Card>
  <CardHeader>
    <CardTitle>Настройка начисления бонусов</CardTitle>
  </CardHeader>
  <CardContent>
    {/* Переключатель режимов */}
    <div className="grid grid-cols-2 gap-4 mb-6">
      <ModeCard
        mode="SIMPLE"
        selected={bonusMode === 'SIMPLE'}
        onClick={() => handleModeChange('SIMPLE')}
        title="Простой режим"
        description="Фиксированный процент для всех"
        icon={<Percent />}
      />
      <ModeCard
        mode="LEVELS"
        selected={bonusMode === 'LEVELS'}
        onClick={() => handleModeChange('LEVELS')}
        title="Уровни бонусов"
        description="Процент зависит от суммы покупок"
        icon={<Target />}
      />
    </div>

    {/* Настройки режима */}
    {bonusMode === 'SIMPLE' ? (
      <SimpleModeSettings />
    ) : (
      <LevelsModeSettings />
    )}
  </CardContent>
</Card>
```

## 🎯 Рекомендуемое решение

### Комбинированный подход (Вариант 1 + улучшения)

#### 1. Добавить поле `bonusMode` в схему
```prisma
model Project {
  // ...
  bonusMode BonusMode @default(SIMPLE) @map("bonus_mode")
}

enum BonusMode {
  SIMPLE  // Фиксированный процент
  LEVELS  // Уровни бонусов
}
```

#### 2. Создать компонент `BonusModeSelector`
```tsx
<BonusModeSelector
  value={bonusMode}
  onChange={handleModeChange}
  hasLevels={bonusLevels.length > 0}
  onConfirmChange={showConfirmDialog}
/>
```

#### 3. Объединить настройки в одном месте
```
Настройки проекта
├── Основная информация
├── Приветственное вознаграждение
└── Начисление бонусов ← НОВЫЙ РАЗДЕЛ
    ├── Режим начисления (переключатель)
    ├── Простой режим (если выбран)
    │   └── Процент бонусов
    └── Уровни бонусов (если выбран)
        ├── Список уровней
        └── Кнопка "Добавить уровень"
```

#### 4. Добавить защиту от потери данных
```tsx
// При переключении с LEVELS на SIMPLE
if (bonusLevels.length > 0) {
  showDialog({
    title: "Переключить на простой режим?",
    description: `У вас настроено ${bonusLevels.length} уровней. 
                  Они будут деактивированы, но не удалены.
                  Вы сможете вернуться к ним позже.`,
    actions: [
      { label: "Отмена", variant: "outline" },
      { label: "Переключить", variant: "default" }
    ]
  });
}
```

#### 5. Визуальные подсказки
```tsx
// В простом режиме
<Alert>
  <Lightbulb className="h-4 w-4" />
  <AlertTitle>Совет</AlertTitle>
  <AlertDescription>
    Хотите мотивировать клиентов делать больше покупок? 
    Попробуйте <Link>уровни бонусов</Link> — начисляйте 
    больший процент постоянным клиентам.
  </AlertDescription>
</Alert>

// В режиме уровней
<Alert>
  <Info className="h-4 w-4" />
  <AlertTitle>Как это работает</AlertTitle>
  <AlertDescription>
    Процент начисления зависит от общей суммы покупок клиента.
    Чем больше покупок — тем выше уровень и процент бонусов.
  </AlertDescription>
</Alert>
```
