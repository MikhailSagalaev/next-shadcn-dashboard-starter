# Интеграция виджета

Общее руководство по подключению бонусной системы Gupil к вашему сайту.

> **Универсальный виджет!** Gupil работает на любой платформе электронной коммерции. Выберите вашу платформу и следуйте инструкциям.

## Как работает виджет?

### Архитектура

```
┌─────────────────────────────────────────────────┐
│           widget-loader.js (Entry Point)        │
│  • Автоопределение платформы                    │
│  • Динамическая загрузка адаптера               │
│  • Инициализация Core                           │
└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐
│         universal-widget.js (Core)              │
│  • Управление состоянием                        │
│  • API запросы к серверу                        │
│  • Бизнес-логика бонусов                        │
│  • UI компоненты                                │
└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐
│          Platform Adapter (Interface)           │
│  • tilda-adapter.js (Tilda)                     │
│  • shopify-adapter.js (Shopify - coming soon)   │
│  • woocommerce-adapter.js (WooCommerce - soon)  │
│  • custom-adapter.js (Fallback)                 │
└─────────────────────────────────────────────────┘
```

### Процесс загрузки

1. **Загрузка Widget Loader**
   - Браузер загружает `widget-loader.js`
   - Loader ждет готовности DOM

2. **Определение платформы**
   - Loader проверяет наличие специфичных объектов/элементов
   - Определяет платформу: Tilda, Shopify, WooCommerce или Custom

3. **Загрузка адаптера**
   - Loader динамически загружает нужный адаптер
   - Например, для Tilda загружается `tilda-adapter.js`

4. **Загрузка Core**
   - Loader загружает `universal-widget.js`
   - Core инициализируется с адаптером

5. **Инициализация виджета**
   - Виджет начинает работу
   - Регистрирует пользователей, отображает бонусы, применяет промокоды

## Быстрый старт

### Шаг 1: Выберите платформу

Определите какую платформу вы используете:

- **Tilda** → [Инструкция для Tilda](/tilda-integration)
- **Shopify** → Coming Soon (Q2 2026)
- **WooCommerce** → Coming Soon (Q2 2026)
- **Другая платформа** → [Инструкция для Custom](/custom-integration)

### Шаг 2: Получите код интеграции

1. Войдите в админ-панель Gupil
2. Перейдите в **Проекты** → Выберите ваш проект
3. Откройте вкладку **Интеграция**
4. Выберите вашу платформу
5. Скопируйте код подключения

### Шаг 3: Добавьте код на сайт

Вставьте код на ваш сайт (обычно в `<head>` или перед `</body>`):

```html
<!-- Gupil Bonus Widget -->
<script>
  window.gupilConfig = {
    projectId: 'YOUR_PROJECT_ID',
    apiUrl: 'https://gupil.ru/api'
  };
</script>
<script src="https://gupil.ru/widget-loader.js" async></script>
```

### Шаг 4: Настройте Webhook

Webhook нужен для синхронизации заказов с Gupil:

1. В Gupil: **Интеграция** → Скопируйте **Webhook URL**
2. В вашей платформе: Настройте отправку webhook при создании заказа
3. Формат: **JSON**, Метод: **POST**

**Готово!** Виджет начнет работать на вашем сайте.

## Конфигурация

### Базовая конфигурация

```javascript
window.gupilConfig = {
  // Обязательные параметры
  projectId: 'YOUR_PROJECT_ID',    // ID проекта из админ-панели
  apiUrl: 'https://gupil.ru/api',  // URL API сервера
  
  // Опциональные параметры
  platform: 'tilda',               // Принудительный выбор платформы
  debug: false,                    // Включить debug логи
  theme: 'light',                  // Тема виджета: light | dark | auto
  language: 'ru',                  // Язык: ru | en
};
```

### Расширенная конфигурация

```javascript
window.gupilConfig = {
  projectId: 'YOUR_PROJECT_ID',
  apiUrl: 'https://gupil.ru/api',
  
  // Платформа
  platform: 'custom',              // Принудительный выбор
  
  // Внешний вид
  theme: 'dark',                   // Темная тема
  language: 'en',                  // Английский язык
  
  // Отладка
  debug: true,                     // Debug логи в консоль
  
  // Кастомные селекторы (для custom платформы)
  selectors: {
    emailInput: '#customer-email',
    phoneInput: '#customer-phone',
    cartTotal: '.cart-total-amount',
    cartItems: '.cart-item'
  },
  
  // Callbacks
  onReady: function(widget) {
    console.log('Виджет готов!', widget);
    // Можно вызывать методы widget
  },
  onError: function(error) {
    console.error('Ошибка виджета:', error);
    // Отправить в систему мониторинга
  }
};
```

## Troubleshooting

### Виджет не загружается

**Симптомы:**
- Виджет не появляется на сайте
- Нет логов в консоли

**Решение:**
1. Откройте консоль браузера (F12)
2. Проверьте вкладку **Network** → `widget-loader.js` должен загрузиться
3. Проверьте вкладку **Console** → должны быть логи `[Gupil Widget]`
4. Убедитесь что `projectId` указан правильно

**Debug режим:**
```javascript
window.gupilConfig = {
  projectId: 'YOUR_PROJECT_ID',
  apiUrl: 'https://gupil.ru/api',
  debug: true // Включить debug логи
};
```

---

### Платформа определяется неправильно

**Симптомы:**
- В консоли: `[Gupil Widget] Platform detected: custom` (но у вас Tilda)
- Виджет работает некорректно

**Решение:**
Принудительно укажите платформу:

```javascript
window.gupilConfig = {
  projectId: 'YOUR_PROJECT_ID',
  apiUrl: 'https://gupil.ru/api',
  platform: 'tilda' // Принудительно использовать Tilda
};
```

---

### Бонусы не отображаются

**Симптомы:**
- Плашки с бонусами не появляются на товарах
- Информация о бонусах не показывается в корзине

**Решение:**
1. Проверьте что виджет загружен (см. "Виджет не загружается")
2. Проверьте настройки проекта в админ-панели:
   - **Проект** → **Настройки** → **Бонусная система** → Включена
   - **Проект** → **Настройки** → **Процент начисления** → Указан
3. Для Tilda: Проверьте что используется поддерживаемый блок каталога
4. Для Custom: Проверьте что селекторы указаны правильно

---

### Промокод не применяется

**Симптомы:**
- При вводе промокода ничего не происходит
- Ошибка в консоли

**Решение:**
1. Проверьте что промокод активен в админ-панели:
   - **Промокоды** → Найдите промокод → Статус: **Активен**
2. Проверьте срок действия промокода
3. Проверьте лимиты использования
4. Проверьте консоль на наличие ошибок API:
```
[Gupil Widget] Error applying promocode: Invalid code
```

---

### Webhook не приходит

**Симптомы:**
- После оформления заказа webhook не приходит в Gupil
- В логах Gupil нет записей о webhook

**Решение:**
1. Проверьте что Webhook URL указан правильно в вашей платформе
2. Проверьте что формат данных: **JSON**
3. Проверьте что метод: **POST**
4. Проверьте что сайт опубликован после настройки webhook
5. Оформите тестовый заказ и проверьте логи в Gupil

**Тест webhook вручную:**
```bash
curl -X POST https://gupil.ru/api/webhook/YOUR_SECRET \
  -H "Content-Type: application/json" \
  -d '{
    "action": "purchase",
    "payload": {
      "email": "test@example.com",
      "amount": 5000,
      "orderId": "TEST-123"
    }
  }'
```

Если тест успешен, но webhook с сайта не приходит - проблема в настройках вашей платформы.

### Ошибка CORS

**Симптомы:**
```
Access to fetch at 'https://gupil.ru/api/...' from origin '...' 
has been blocked by CORS policy
```

**Решение:**
Это проблема на стороне сервера. Обратитесь в поддержку Gupil: support@gupil.ru

### Виджет конфликтует с другими скриптами

**Симптомы:**
- Виджет не работает
- Ошибки JavaScript в консоли
- Другие скрипты на сайте перестали работать

**Решение:**
1. Загружайте виджет последним (перед `</body>`)
2. Используйте `async` атрибут для асинхронной загрузки
3. Проверьте консоль на наличие конфликтов

**Пример правильной загрузки:**
```html
<!-- Другие скрипты -->
<script src="jquery.js"></script>
<script src="bootstrap.js"></script>

<!-- Gupil Widget (последним) -->
<script>
  window.gupilConfig = { /* ... */ };
</script>
<script src="https://gupil.ru/widget-loader.js" async></script>
</body>
```

## FAQ

### Можно ли использовать виджет на нескольких сайтах?

Да, один проект может использоваться на нескольких доменах. Укажите все домены в настройках проекта в админ-панели.

### Как обновить виджет до новой версии?

Если используете CDN (`https://gupil.ru/widget-loader.js`) - обновление происходит автоматически. Если self-hosted - скачайте новые файлы и замените старые.

### Поддерживается ли мобильная версия?

Да, виджет полностью адаптивен и работает на всех устройствах (desktop, tablet, mobile).

### Можно ли кастомизировать дизайн виджета?

Да, через CSS переменные или кастомные стили. Подробнее в документации для разработчиков.

### Влияет ли виджет на скорость загрузки сайта?

Минимально. Виджет загружается асинхронно (`async`) и не блокирует рендеринг страницы. Размер файлов:
- `widget-loader.js`: ~5 KB (gzip)
- `universal-widget.js`: ~15 KB (gzip)
- `tilda-adapter.js`: ~10 KB (gzip)

**Итого:** ~30 KB (меньше одной картинки).
