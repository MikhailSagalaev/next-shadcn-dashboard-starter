## Шаг 1: Получение API credentials в InSales

### 1.1 Войти в админ-панель InSales
```
https://[ваш-магазин].myinsales.ru/admin
```

### 1.2 Создать API ключ
1. Перейдите в **Настройки** → **API**
2. Нажмите **"Создать новый API ключ"**
3. Заполните форму:
   - **Название:** "Gupil Bonus System"
   - **Права доступа:** 
     - ✅ Чтение заказов
     - ✅ Чтение клиентов
     - ✅ Чтение товаров (для виджета)
4. Нажмите **"Создать"**

### 1.3 Сохранить credentials
После создания вы получите:
- **API Key** (ID приложения) - например: `5d7533619e1a01669d8b2e5a50b1e82a`
- **API Password** (Пароль приложения) - например: `31ef1322e03069b9387180b67aebc2f7`

⚠️ **ВАЖНО:** API Password показывается только один раз! Сохраните его в безопасном месте.

## Шаг 4: Встраивание виджета на сайт

### 4.1 Скопировать код виджета
В Gupil dashboard в разделе **"Код виджета для сайта"** скопируйте HTML код:

```html

<script 
  src="https://gupil.ru/insales-widget-loader.js" 
  data-project-id="cmilhq0y600099e7uraiowrmt"
></script>
```

### 4.2 Вставить код в тему InSales
1. Перейдите в **Дизайн** → **Редактор тем**
2. Откройте файл `layout.liquid` или `theme.liquid`

## Шаг 5: Тестирование интеграции

### 5.1 Тест 1: Создание клиента

**Действие:**
1. Откройте магазин в режиме инкогнито
2. Зарегистрируйтесь как новый клиент
3. Укажите email: `test-insales@example.com`

**Ожидаемый результат:**
- ✅ Клиент создан в InSales
- ✅ Webhook `clients/create` отправлен
- ✅ Пользователь создан в Gupil
- ✅ Начислены приветственные бонусы (если настроено в проекте)

**Проверка в Gupil:**
```
https://gupil.ru/dashboard/projects/cmilhq0y600099e7uraiowrmt/users
```
Найдите пользователя с email `test-insales@example.com`

**Проверка логов:**
```
https://gupil.ru/dashboard/projects/cmilhq0y600099e7uraiowrmt/integrations/insales
```
Прокрутите вниз до раздела **"Логи Webhooks"** - должна быть запись `clients/create` со статусом "Успешно"

### 5.2 Тест 2: Создание заказа (без использования бонусов)

**Действие:**
1. Войдите под созданным клиентом
2. Добавьте товар в корзину (например, на 5000₽)
3. Оформите заказ
4. Оплатите заказ (или измените статус на "Оплачен" в админке)

**Ожидаемый результат:**
- ✅ Webhook `orders/create` отправлен
- ✅ Начислены бонусы: 5000₽ × 10% = 500 бонусов
- ✅ Создана транзакция типа EARN
- ✅ Баланс пользователя увеличился на 500

**Проверка в Gupil:**
1. Откройте профиль пользователя
2. Проверьте баланс бонусов: должно быть 500 (+ приветственные, если были)
3. Проверьте историю транзакций: должна быть запись "Покупка #XXX"

**Проверка в виджете:**
1. Откройте магазин под этим клиентом
2. Кликните на виджет бонусов
3. Баланс должен показывать 500 бонусов

### 5.3 Тест 3: Использование бонусов при заказе

**Действие:**
1. Добавьте товар на 2000₽ в корзину
2. На странице оформления заказа примените 200 бонусов
3. Оплатите заказ

**Ожидаемый результат (режим SPEND_AND_EARN):**
- ✅ Списано 200 бонусов
- ✅ Начислено: (2000₽ - 200₽) × 10% = 180 бонусов
- ✅ Итоговый баланс: 500 - 200 + 180 = 480 бонусов

**Проверка:**
1. Баланс в профиле: 480 бонусов
2. Две транзакции:
   - SPEND: -200 "Оплата заказа #XXX"
   - EARN: +180 "Покупка #XXX"

**Действие:**
Откройте страницу интеграции и проверьте логи webhooks

**Ожидаемый результат:**
- ✅ Все webhooks имеют статус "Успешно" (зеленый бейдж)
- ✅ HTTP статус: 200
- ✅ При раскрытии лога видны payload и response
- ✅ Нет ошибок в поле "error"

### 5.5 Тест 5: Статистика интеграции

**Проверка карточек статистики:**
- **Всего заказов:** 2
- **Бонусов начислено:** 680 (500 + 180)

## 🐛 Troubleshooting

### Проблема 1: Webhook не приходит
**Симптомы:** Заказ создан, но бонусы не начислены

**Решение:**
1. Проверьте URL webhook в InSales - должен быть точно `https://gupil.ru/api/insales/webhook/[projectId]`
2. Проверьте, что webhook активен (галочка)
3. Проверьте логи InSales - там должен быть статус ответа
4. Проверьте логи сервера: `pm2 logs bonus-app | grep webhook`

### Проблема 2: Бонусы не начисляются
**Симптомы:** Webhook приходит, но бонусы = 0

**Решение:**
1. Проверьте статус оплаты заказа - должен быть `paid`
2. Проверьте процент начисления в настройках интеграции
3. Проверьте логи webhook - там должно быть `bonusAwarded: X`
4. Проверьте режим BonusBehavior проекта

### Проблема 3: Виджет не отображается
**Симптомы:** Код вставлен, но виджета нет

**Решение:**
1. Откройте консоль браузера (F12) - проверьте ошибки JavaScript
2. Проверьте, что скрипт загружается: `https://gupil.ru/insales-widget-loader.js`
3. Проверьте `data-project-id` - должен совпадать с ID проекта
4. Проверьте, что интеграция активна (`isActive: true`)

### Проблема 4: Дублирование начисления
**Симптомы:** За один заказ начислено дважды

**Решение:**
1. Проверьте логи - должна быть проверка на дубликаты
2. Если дубликаты есть - проверьте webhook `orders/update`
3. Отключите webhook `orders/update` если не нужен

### Проблема 5: Ошибка 500 при создании интеграции
**Симптомы:** Форма не сохраняется, ошибка 500

**Решение:**
1. Проверьте логи: `pm2 logs bonus-app --err`
2. Проверьте, что выполнен `npx prisma generate`
3. Проверьте, что все поля заполнены корректно
4. Проверьте формат Shop Domain: `shop.myinsales.ru` (без https://)
