# Проблема с импортом Workflow

## Описание проблемы

При импорте workflow в конструкторе показывается только триггер `/start`, остальные ноды не видны.

## Причина

Это **нормальное поведение** конструктора workflow. Вот почему:

### Как работает визуализация workflow

1. **При импорте** показываются только **триггеры** (точки входа)
2. **Остальные ноды** появляются только когда:
   - Вы кликаете на триггер и "разворачиваете" его
   - Workflow выполняется и создается execution log
   - Вы вручную добавляете ноды через интерфейс

### Это не баг, это фича!

Конструктор скрывает неиспользуемые ноды, чтобы:
- Не загромождать canvas
- Показывать только активные пути выполнения
- Упростить навигацию в сложных workflow

## Как проверить, что workflow импортирован правильно

### 1. Проверка через код
```powershell
$json = Get-Content "Система лояльности без подписки (шаблон).json" -Raw | ConvertFrom-Json

Write-Host "Количество нод: $($json.nodes.Count)" -ForegroundColor Green
Write-Host "Количество соединений: $($json.connections.Count)" -ForegroundColor Green

# Проверим ключевые ноды
$keyNodes = @("start-trigger", "welcome-message", "check-contact-user", "activate-user", "success-activated-user")

foreach ($nodeId in $keyNodes) {
    $exists = $json.nodes | Where-Object { $_.id -eq $nodeId }
    $status = if ($exists) { "✅" } else { "❌" }
    Write-Host "$status $nodeId" -ForegroundColor $(if ($exists) { "Green" } else { "Red" })
}
```

### 2. Проверка через UI

После импорта:

1. **Кликните на триггер** `/start` на canvas
2. **Нажмите "Выполнить"** или "Test"
3. **Посмотрите execution log** - там будут все выполненные ноды
4. **Или откройте JSON view** в конструкторе (если есть такая опция)

### 3. Тестирование workflow

Лучший способ проверить - **запустить бота** и протестировать:

```bash
# 1. Убедитесь что бот запущен
# 2. Отправьте /start боту
# 3. Проверьте, что приходит welcome message
# 4. Поделитесь контактом
# 5. Проверьте, что аккаунт активирован и бонусы начислены
```

## Что делать, если workflow не работает

### 1. Проверьте логи бота

```powershell
# Посмотрите логи выполнения workflow
# В dashboard -> Projects -> [ваш проект] -> Workflow -> Executions
```

### 2. Проверьте настройки проекта

- `welcomeBonus` > 0
- `bonusExpiryDays` установлен (например, 365)
- `bonusMode` = SIMPLE или LEVELS
- `bonusPercentage` > 0 (для SIMPLE режима)

### 3. Проверьте Telegram бота

- Бот запущен и работает
- Webhook настроен правильно
- Bot token корректный
